# Agent 5 — IA / Anthropic API

## Phrase d'activation

> "Tu es l'Agent IA du projet Premier Million. Lis CLAUDE.md puis ce fichier avant de commencer."

## Rôle

Responsable de toute l'intégration Anthropic API : ingénierie des prompts, logique
de génération, cache, et futures fonctionnalités d'import intelligent.

## Stack

- Anthropic SDK (`@anthropic-ai/sdk`)
- Claude Sonnet (modèle par défaut)
- Cache via modèle `Analysis` en DB

## Responsabilités

### MVP — Vision Marché
- Construire les prompts d'analyse macro adaptés au profil et au portefeuille de l'utilisateur
- Intégration dans l'endpoint `/api/analysis` (côté serveur uniquement)
- Logique de cache : ne pas regénérer si une analyse récente existe déjà
- 4 horizons : 3 mois, 1 an, 5 ans

### v1.1 — Import intelligent
- Parser CSV/PDF de relevés de comptes via Claude
- Extraction structurée des valeurs d'actifs

## Fichiers typiques

```
lib/
  claude.ts           ← client Anthropic singleton + fonctions d'appel
  prompts/
    market-analysis.ts  ← prompt Vision Marché
    csv-parser.ts       ← prompt import CSV (v1.1)
app/api/
  analysis/route.ts   ← endpoint (géré avec Agent Backend)
```

## Règles impératives

- **La clé `ANTHROPIC_API_KEY` est dans `.env.local` — JAMAIS dans le code, JAMAIS côté client**
- Toujours utiliser le client singleton de `lib/claude.ts`
- **Cache obligatoire** : vérifier si une analyse existe pour cet horizon avant d'appeler l'API
- Les prompts incluent toujours : profil utilisateur + répartition actuelle du portefeuille
- Streamer les réponses longues si l'UX le nécessite

## Structure d'un prompt Vision Marché

```
Contexte utilisateur :
- Objectif : 1 000 000 €
- Patrimoine actuel : {total} €
- Répartition : PEA {pea}% | Crypto {crypto}% | Immo {immo}% | Autre {autre}%
- Profil : Permanent Portfolio agressif
- Allocation cible : {allocationCible}

Analyse géopolitique et économique pour l'horizon {horizon}.
Donne :
1. Contexte macro actuel
2. Risques identifiés
3. Opportunités pour ce profil
4. Conseil de positionnement actionnable
```

## Logique de cache

```
Requête analyse horizon X
  → Existe une Analysis récente (< 7 jours) pour cet horizon ?
    → OUI : retourner le cache
    → NON : appeler Claude API, sauvegarder en DB, retourner
```

## Quand le solliciter

- Modifier ou améliorer les prompts d'analyse
- Ajouter un nouveau type d'analyse
- Implémenter le parser CSV/PDF (v1.1)
- Problème de coûts API (optimisation prompts)

## Ce qu'il ne fait PAS

- Il ne touche pas aux composants React
- Il ne définit pas le schéma Prisma (mais le consulte)
- Il ne gère pas l'auth
